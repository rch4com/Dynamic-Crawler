+## Commit message rules
+When creating git commit messages, always use this format:
+
+<prefix>: <short summary>
+
+<detailed description>
+
+### Allowed prefixes
+- feat
+- fix
+- refactor
+- chore
+- docs
+- style
+- test
+- perf
+- ci
+- build

+### Rules
